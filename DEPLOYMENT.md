# WAZA - Guide de Deploiement Production

## Architecture de Deploiement

```
Frontend (Vercel)          Backend (Railway)          Worker (Railway)
https://waza.vercel.app    https://waza-backend       celery worker
       |                   .up.railway.app             (same codebase)
       |                          |
       +--------> API calls ------+
                                  |
                          PostgreSQL (Railway Plugin)
                          Redis (Railway Plugin)
```

---

## 1. Backend - Railway.app

### Etape 1 : Creer un projet Railway

1. Aller sur https://railway.app et se connecter
2. Cliquer **New Project** > **Deploy from GitHub repo**
3. Selectionner le repository WAZA
4. Railway detecte automatiquement le `railway.json`

### Etape 2 : Ajouter PostgreSQL

1. Dans le projet Railway, cliquer **+ New** > **Database** > **Add PostgreSQL**
2. Railway cree automatiquement la variable `DATABASE_URL`
3. Verifier dans **Variables** que `DATABASE_URL` est bien definie

### Etape 3 : Ajouter Redis

1. Cliquer **+ New** > **Database** > **Add Redis**
2. Railway cree automatiquement la variable `REDIS_URL`
3. Verifier dans **Variables** que `REDIS_URL` est bien definie

### Etape 4 : Configurer les Variables d'Environnement

Dans le service backend Railway, aller dans **Variables** et ajouter :

```
DATABASE_URL=<auto-generated by Railway PostgreSQL plugin>
REDIS_URL=<auto-generated by Railway Redis plugin>
JWT_SECRET_KEY=waza-super-secret-key-massudom-2026
EMERGENT_LLM_KEY=<your Emergent LLM universal key>
WHATSAPP_APP_SECRET=mock_secret
WHATSAPP_VERIFY_TOKEN=waza_verify_token_2026
CINETPAY_API_KEY=mock_cinetpay_key
CINETPAY_SITE_ID=mock_site_id
STRIPE_API_KEY=<your Stripe test secret key>
STRIPE_WEBHOOK_SECRET=mock_webhook_secret
FRONTEND_URL=https://waza.vercel.app
ENVIRONMENT=production
CORS_ORIGINS=https://waza.vercel.app,http://localhost:3000
```

### Etape 5 : Deployer

Railway deploie automatiquement. La commande de demarrage est :
```
cd backend && alembic upgrade head && uvicorn server:app --host 0.0.0.0 --port $PORT
```

### Etape 6 : Verifier

```bash
# Health check
curl https://waza-backend.up.railway.app/health
# Expected: {"status":"ok"}

# API Health
curl https://waza-backend.up.railway.app/api/health
# Expected: {"status":"healthy","service":"waza-backend",...}

# Plans
curl https://waza-backend.up.railway.app/api/billing/plans
# Expected: Liste des 4 plans (free, starter, pro, business)

# Login
curl -X POST https://waza-backend.up.railway.app/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"demo@waza.africa","password":"Password123!"}'
# Expected: {"access_token":"...","refresh_token":"...","user":{...}}
```

## 2. Celery Worker - Railway (Service Separe)

### Etape 1 : Creer un nouveau service

1. Dans le meme projet Railway, cliquer **+ New** > **Empty Service**
2. Connecter le meme repository GitHub
3. Configurer le **Start Command** :
```
cd backend && celery -A tasks.celery_app worker --loglevel=info
```

### Etape 2 : Variables d'Environnement

Copier les memes variables que le backend, surtout :
- `DATABASE_URL` (reference au plugin PostgreSQL)
- `REDIS_URL` (reference au plugin Redis)
- `EMERGENT_LLM_KEY`
- `WHATSAPP_APP_SECRET`

## 3. Frontend - Vercel

### Etape 1 : Creer un projet Vercel

1. Aller sur https://vercel.com et se connecter
2. Cliquer **Add New** > **Project**
3. Importer le repository GitHub
4. Configurer :
   - **Framework Preset**: Create React App
   - **Root Directory**: `frontend`
   - **Build Command**: `yarn build`
   - **Output Directory**: `build`

### Etape 2 : Variables d'Environnement

Dans les settings Vercel, ajouter :
```
REACT_APP_BACKEND_URL=https://waza-backend.up.railway.app
```

> Note : En production avec Vercel + CRA, utiliser `REACT_APP_` prefix.
> Si migration vers Vite, utiliser `VITE_API_URL`.

### Etape 3 : Deployer

Vercel deploie automatiquement a chaque push sur `main`.

### Etape 4 : Verifier

1. Ouvrir `https://waza.vercel.app`
2. La landing page WAZA doit s'afficher
3. Se connecter avec `demo@waza.africa` / `Password123!`
4. Le dashboard doit s'ouvrir avec les stats

## 5. Domaines Personnalises (Optionnel)

### Backend : api.waza.africa
1. Dans Railway > Service > Settings > Custom Domain
2. Ajouter `api.waza.africa`
3. Configurer DNS : CNAME vers l'URL Railway

### Frontend : waza.africa
1. Dans Vercel > Project > Settings > Domains
2. Ajouter `waza.africa` et `app.waza.africa`
3. Configurer DNS : CNAME vers `cname.vercel-dns.com`
4. SSL automatique sur les deux

Apres ajout des domaines personnalises, mettre a jour :
- `FRONTEND_URL=https://waza.africa` dans Railway
- `CORS_ORIGINS=https://waza.africa,https://app.waza.africa` dans Railway
- `REACT_APP_BACKEND_URL=https://api.waza.africa` dans Vercel
